# Conversation Workbench for ChatGPT 🧰✨

🧩 一个专注于优化长篇 ChatGPT 对话体验的浏览器扩展，通过整理对话降低卡顿，并提供时间线导航、搜索、导出、提示词管理、文件夹管理与偏好设置能力。  

🧩 A browser extension focused on improving long ChatGPT conversations by reducing lag through conversation cleanup, with timeline navigation, search, export, prompt management, folder management, and preference settings.

## ✨ 简介 | Overview 🌟

### 这个项目解决什么问题？ 🤔💬

如果你经常把 ChatGPT 当成工作台来用，你大概率会遇到这些问题：

- 🐢 对话一长就越来越卡，滚动和浏览体验明显变差
- 🧭 历史内容越来越难找，前文后文来回翻很费劲
- 📍 缺少直观的导航方式，定位某一段对话不够顺手
- 📦 导出会话、复用提示词、管理工作流比较零散
- 🧠 在深度思考拆段、图片生成、多段回复等场景下，结构和阅读体验容易割裂

**Conversation Workbench for ChatGPT** 的目标，就是把这些“长对话工作流问题”系统化地补上。

### 与原项目相比，这个项目做了哪些重构与增强？ 🔧🚀

本项目并不是对原始实现做少量界面调整或功能拼接，而是在原项目启发基础上进行了**大幅度重构与扩展**。核心变化包括：

- 🧠 **底层语义重构**：从更偏 DOM / message 级处理，演进为更清晰的 `message → turn → QA group` 三层模型，使整理、导航、搜索定位和自动整理都围绕更稳定的语义单位工作。
- 🚀 **长对话体验增强**：针对长对话卡顿、阅读困难和定位低效等问题，重新设计了对话整理、组级时间线导航和搜索定位逻辑。
- 🧩 **复杂场景兼容增强**：对深度思考拆段、图片创建、多段 assistant 回复等场景做了更完整的适配，减少“看得到但定位不稳”或“结构正确但体验割裂”的问题。
- 📚 **功能范围扩展**：新增或系统化整合了偏好设置、自动整理、提示词库、组级导航、导出、文件夹管理与本地化等能力。
- 🎨 **界面与产品语言重做**：不再沿用原始工具面板风格，而是以更完整的中文产品化界面、命名、层级和视觉系统重构整体体验。
- 🖱️ **交互体验增强**：悬浮图标与面板支持更灵活的拖动与位置持久化，悬浮入口与工具面板可同时存在，整体更接近可长期使用的工作台体验。

因此，本项目更适合被理解为一个**独立维护的衍生改进版本**，而不是对原项目的简单换皮。

### 适合谁使用？ 👀🎯

这个项目尤其适合：

- 💼 重度 ChatGPT 用户
- 🧵 经常处理长对话、长线程的人
- 📍 需要快速定位上下文的人
- 🔍 需要搜索、导航、导出和提示词复用的人
- 🛠️ 希望把 ChatGPT 当作资料台 / 工作台 / 长线程协作工具使用的人

## 🚀 安装与使用 | Installation & Quick Start 🛠️

### 支持站点 🌐

- 💬 `https://chat.openai.com/*`
- 🤖 `https://chatgpt.com/*`

### 支持浏览器 🧭

- 🌈 Chrome
- 🌐 Edge
- 🦊 Firefox（临时加载方式）

### Chrome 安装 🌈

1. 打开 `chrome://extensions/`
2. 打开右上角 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择当前项目根目录

### Edge 安装 🌐

1. 打开 `edge://extensions/`
2. 打开右上角 **开发人员模式**
3. 点击 **加载已解压的扩展**
4. 选择当前项目根目录

1. 打开 `about:debugging#/runtime/this-firefox`
2. 点击 **临时载入附加组件**
3. 选择当前项目根目录下的 `manifest.json`

## ▶️ 使用说明 | How to Use 📝

### 1. 打开工具栏 🧰

页面加载后，页面侧边会显示 **ChatGPT 工具** 面板。  
收起后会变成一个可点击、可拖动、可贴边的悬浮图标；点击图标可再次打开面板，且图标本身仍会保留在页面中，作为稳定入口。

### 2. 整理对话 🧹

点击 **整理对话** 后，扩展会隐藏较早内容，仅保留最近的一部分对话，以降低长对话页面卡顿感、提升浏览流畅度。  
这里的保留与整理逻辑基于更稳定的 **QA group** 语义，而不是简单的零散 message 数量。

### 3. 展开全部 ♻️

点击 **展开全部** 后，会把之前隐藏的内容恢复到页面中，方便完整阅读或继续回查。  
恢复时会尽量保持当前阅读位置，避免页面突然跳到顶部。

### 4. 使用对话导航 🕒

打开 **对话导航** 后，可通过时间线快速定位到某一组对话。  
时间线支持：

- 👀 节点预览
- 🎯 点击跳转
- ✨ 当前节点高亮
- 🖱️ 滚轮滚动
- 🧲 拖拽移动
- 🧩 按 QA group 组织导航节点，而不是简单按单条消息打点

### 5. 搜索当前对话 🔍

在搜索区输入关键词后：

- 🔎 可在当前对话中搜索匹配内容
- ✨ 支持高亮
- ↕️ 支持上一条 / 下一条跳转
- 🧩 富媒体、图片创建、多段回复等特殊场景下，会优先保证稳定定位到对应对话组
- 📌 搜索结果仍保留 message 命中精度，但定位语义更接近 QA group

### 6. 导出会话 📦

点击 **导出会话** 后，可将当前对话导出为 JSON。  
即使之前已经执行过整理，导出仍会尽量保留完整会话内容。

### 7. 使用提示词库 📚

点击 **提示词库** 可打开本地提示词管理面板，支持：

- ➕ 新增
- 🗑️ 删除
- 🔍 搜索
- 🗂️ 分类
- ↕️ 排序
- 📥 导入 JSON
- 📤 导出 JSON
- 📋 一键复制

### 8. 使用偏好设置 ⚙️

点击 **偏好设置** 可配置当前插件的关键行为，例如：

- 🤖 自动整理开关
- 📏 自动整理阈值
- 🧾 保留最近 QA 组数
- 🕒 时间线最大节点数
- 🌐 语言切换（含自动跟随浏览器）

### 9. 使用文件夹管理 📁

侧边栏中的文件夹功能支持本地对话归类与整理。  
在当前版本中，文件夹相关界面也已统一到同一套产品化视觉风格下，支持：

- ➕ 新建文件夹
- ✏️ 重命名
- 🗑️ 删除
- 📂 折叠 / 展开
- 🧲 拖拽归类
- ↕️ 排序调整
- 💾 刷新后恢复本地结构

## 🧰 功能介绍 | Features ✨

### 1. 整理对话（降低长对话卡顿） 🧹

这是本项目最核心的功能之一。  
它的目标不是“删掉历史”，而是通过隐藏较早内容、保留最近内容，让长篇对话在浏览时更顺滑。

适合场景：

- 🐢 对话已经很长，滚动明显发卡
- 🎯 只想先专注最近几轮上下文
- ⚡ 需要先提高浏览流畅度，再决定是否展开全部

### 2. 对话导航（Timeline / Group Navigation） 🕒

对话导航不是简单地给 message 打点，而是围绕更稳定的对话语义单位生成导航节点。  
在当前版本中，它更偏向于按 **QA group** 来组织导航，从而让“一个问题 + 对应回复”更接近用户直觉。

特点：

- ⚡ 支持快速跳转
- 👀 支持节点预览
- ✨ 支持当前节点跟随滚动高亮
- 🧲 支持拖拽与位置调整
- 🧵 更适合长线程浏览

### 3. 搜索当前对话 🔍

搜索功能支持：

- 🔎 关键词匹配
- ✨ 文本高亮
- ↔️ 前后跳转
- 🧩 与整理 / 导航配合使用

针对深度思考拆段、图片生成等复杂场景，当前版本也做了更稳定的适配，尽量避免“找到文字但跳错位置”的问题。

### 4. 导出会话 📦

可将当前对话导出为 JSON，便于：

- 🗃️ 本地归档
- 📊 后续分析
- 🧾 数据整理
- 🔁 复盘与二次处理

### 5. 提示词库 📚

提示词库用于管理你常用的 Prompt 模板，支持本地增删改查和 JSON 导入导出。  
适合：

- ✍️ 写作模板
- 💻 代码评审模板
- 📄 汇报 / 总结模板
- 🔬 研究与分析模板

### 6. 偏好设置 ⚙️

偏好设置让插件从“固定功能”升级成“可配置功能”。  
你可以根据自己的使用习惯调整：

- ✅ 是否开启自动整理
- ⏱️ 何时自动整理
- 🧩 整理后保留多少组对话
- 🕘 时间线最多显示多少个节点
- 🌍 当前语言展示方式

### 7. 自动整理 🤖

自动整理会在合适条件下自动执行整理逻辑，减少手动点击次数。  
当前思路更偏向于：

- 🙋 用户主动开启后才生效
- 🎯 达到阈值后自动触发
- 🔁 单会话避免重复自动触发
- 🛑 尽量避免在回答流式生成阶段过早打断体验
- ♻️ 手动展开全部后，不会在同一会话中立即再次自动整理

### 8. 对话文件夹 📁

对话文件夹功能会在侧边栏中提供额外的本地管理能力，支持：

- ➕ 新建
- ✏️ 重命名
- 🗑️ 删除
- 📂 折叠 / 展开
- 🧲 拖拽归类
- ↕️ 排序调整
- 💾 刷新后恢复本地结构

它不会替换 ChatGPT 原生会话节点，而是在原有基础上增加本地分类层。

### 9. 多语言与主题同步 🌐🎨

当前支持：

- 🇨🇳 简体中文
- 🇺🇸 English
- 🌐 自动（跟随浏览器）

同时，工具栏、时间线、提示词库、偏好设置和相关面板会跟随 ChatGPT 页面明暗主题变化。

### 10. 悬浮图标与可拖动工作台 🪄

当前版本中的工具入口不再只是单一收起按钮，而是更完整的可拖动工作台交互：

- 🖱️ 面板可拖动
- 📍 悬浮图标可拖动并吸附到页面上下左右边缘
- 💾 图标与面板位置支持持久化
- 📌 图标与面板可以同时存在，不会因打开面板而自动消失

## 🏗️ 技术架构概览 | Architecture Overview 🧠

> 以下内容更偏向开发者阅读 👨‍💻

本项目的关键设计之一，是将会话语义分为三层：

### 1. `message` 💬

最底层的消息单元，用于文本抽取、命中匹配、导出等底层能力。

### 2. `turn` 🔄

中间层语义，更接近页面结构和一轮消息组织。

### 3. `QA group` 🧩

更上层、也更接近用户理解的语义单位。  
整理、导航、搜索定位、自动整理等功能，会更多围绕这一层来组织。

这种设计的意义在于：

- 🔗 避免直接强耦合页面 DOM
- 🧭 让导航和整理逻辑更稳定
- 🧩 让图片生成、深度思考拆段、多段回复等复杂场景更容易适配

## ⚠️ 已知问题 / 注意事项 | Known Limitations 🚧

- 🕒 时间线只对当前页面**已加载**的内容有效，不会主动请求 ChatGPT 尚未渲染的更早历史消息。
- 📁 文件夹管理依赖当前侧边栏 DOM 结构，本地保存分类关系，不会同步到 ChatGPT 服务端。
- 🧱 ChatGPT 页面结构可能会持续变化，未来若 DOM 调整较大，选择器与挂载点可能需要继续适配。
- 🧪 多模态、图片生成、深度思考等复杂场景虽然已增强兼容，但仍建议在重要数据导出前自行检查结果。
- 🧭 拖动位置持久化依赖当前页面视窗与布局，若页面结构或视口尺寸变化较大，位置可能会在极端场景下做安全回退。
